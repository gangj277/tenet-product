# Twitter/X Threads (Ready-to-Use)

> Academic Twitter outreach for Founding Researcher recruitment.

---

## Pre-Launch Tweets (Week 1-2, Before Thread)

### Tweet 1: Problem Awareness
```
The hardest part of a literature review isn't finding papers.

It's the moment you've read 30+ papers and someone asks
"so what does the evidence actually say?" and you freeze.

That synthesis gap — from individual papers to structured
understanding — is where most researchers get stuck.
```

### Tweet 2: Observation
```
I talked to 20+ PhD students about their lit review process.

The pattern is always the same:
- Notes in 4 different tools
- Contradictions tracked mentally (or not at all)
- "I know I read something about X but can't find it"
- Weeks spent reading, can't articulate the consensus

No tool is designed for the synthesis step.
```

### Tweet 3: Building in Public
```
Building something for this.

A research workspace where:
- Every claim maps to a source + page + quote
- Contradictions between papers are surfaced explicitly
- Evidence has confidence levels (not all studies are equal)
- You get claims, gaps, and next-steps — not a blob of text

More soon.
```

## Main Recruitment Thread (Week 3-4)

### Tweet 1 (Hook)
```
I built an AI research workspace that surfaces what literature
reviews usually miss: contradictions, weak evidence, and gaps.

Looking for 30-50 PhD researchers to help shape it.

```

### Tweet 2 (Problem)
```
The problem:

Researchers read 30+ papers for a lit review. They take notes.
They write summaries. But they miss:

- Where Source A directly contradicts Source B
- Which claims have strong vs. weak evidence
- What questions the literature doesn't answer

Because no tool is built for cross-paper synthesis.
```

### Tweet 3 (What I Tried)
```
Current tools are great for parts of the process:

- Elicit → finding and extracting from papers
- Zotero → organizing references
- NotebookLM → asking questions about uploaded docs
- Consensus → yes/no answers from research

But none of them synthesize across sources with structure
and evidence traceability.
```

### Tweet 4 (The Insight)
```
The insight that led to Lumen:

Synthesis isn't a summary. It's structured judgment.

You need:
- Canonical claims ranked by evidence strength
- Contradictions explicitly surfaced
- Methodology quality assessed
- Gaps identified

Not a paragraph. A map.
```

### Tweet 5 (Product Introduction)
```
So I built Lumen.

Give it a research question (+ optional PDFs) and it:

1. Discovers sources across OpenAlex, Semantic Scholar, arXiv
2. Infers your research perspective before searching
3. Extracts evidence with confidence levels
4. Surfaces contradictions between sources
5. Produces 5 structured artifacts

[Screenshot of workspace]
```

### Tweet 6 (Contradiction Feature)
```
The feature researchers care about most: contradiction surfacing.

Lumen actively finds where your sources disagree.

Not "Source A says X and Source B says X too."
Instead: "Source A claims X (high confidence), but Source B
found the opposite using a different methodology."

[Screenshot of evidence map]
```

### Tweet 7 (Evidence Traceability)
```
Every claim in Lumen traces to:
- The specific source
- The page or section
- A direct quote
- A confidence level (high/medium/low)

No hallucinated citations. No vague attributions.

If Lumen can't trace a claim, it flags it as uncertain.
```

### Tweet 8 (Workspace Agent)
```
Lumen also has a workspace agent you can talk to.

Ask it:
- "/devils-advocate" — argue against your synthesis
- "/methodology-critic" — assess evidence quality
- "What contradictions did you find?"
- "What would strengthen the case for X?"

It proposes edits as inline diffs. You accept or reject.
```

## Accounts to Engage With Before Posting

Build genuine engagement with these types of accounts for 2 weeks before the thread:

| Account Type | How to Engage |
|-------------|---------------|
| PhD students tweeting about their research process | Reply with genuine insights about their methodology |
| Researchers sharing lit review tips | Add to the conversation with your experience |
| People discussing AI research tools | Share honest takes (both positive and critical) |
| Systematic review methodology accounts | Engage with their content substantively |

**Rule**: Never reply with "I'm building something for this" during the warmup phase. Just be a useful person in the conversation.
