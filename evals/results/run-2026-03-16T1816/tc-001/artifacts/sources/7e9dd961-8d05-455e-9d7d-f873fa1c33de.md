# COLLAPSE OR THRIVE? PERILS AND PROMISES OF SYNTHETIC DATA IN A SELF-GENERATING WORLD

**Publication:** Paper under double-blind review

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on data produced by previous models degrade in performance. The authors contrast two scenarios: "replace," where models are trained only on synthetic data from the preceding model, leading to collapse, and "accumulate," where all real and synthetic data are retained, avoiding collapse. They extend this analysis to three new generative modeling settings (multivariate Gaussian, kernel density estimation, and supervised finetuning of language models), finding consistent results. A "compromise" scenario with a fixed compute budget is also explored, showing performance plateaus rather than divergence. Finally, the paper examines the interaction between the quantity and proportion of real data, revealing that synthetic data can be beneficial when real data is scarce but detrimental when real data is plentiful.

## Key Claims

*   Model collapse occurs when generative models are trained on successively generated synthetic data in a "replace" scenario (i.e., older data is discarded).
*   Model collapse is avoided in an "accumulate" scenario where all real and synthetic data are retained for subsequent training.
*   These contrasting outcomes hold across multivariate Gaussian modeling, kernel density estimation, and supervised finetuning of language models.
*   In a "compromise" scenario with a fixed compute budget, model performance deteriorates more than in the "accumulate" scenario but plateaus, unlike the divergence seen in the "replace" scenario.
*   The value of synthetic data depends on the quantity of real data: it can improve performance when real data is scarce but harm performance when real data is abundant.
*   Both the cardinality (absolute number) and proportion of real data significantly impact test loss.

## Relevant Evidence/Quotes

*   "First, collapse versus avoidance of collapse is studied by comparing the replace and accumulate scenarios on each of three prominent generative modeling settings; we find the same contrast emerges in all three settings."
*   "Second, we study a compromise scenario; the available data remains the same as in the accumulate scenario – but unlike accumulate and like replace, each model is trained under a fixed compute budget; we demonstrate that model test loss on real data is larger than in the accumulate scenario, but apparently plateaus, unlike the divergence seen with replace."
*   "Third, we study the relative importance of cardinality and proportion of real data for avoiding model collapse. Surprisingly, we find a non-trivial interaction between real and synthetic data, where the value of synthetic data for reducing test loss depends on the absolute quantity of real data."
*   "In our experiments, when the number of real data is 1024 or lower, we find that there is an small but non-zero amount of synthetic data that improves the test loss when it is included. This suggests that practitioners fine-tuning with insufficient amounts of real data should consider supplementing with synthetic data to improve model quality."
*   "On the other hand, when real data are plentiful, we find that more synthetic data almost always harms final model quality when the number of real data is held constant."

## Contribution to Research Frame

This paper directly addresses the core of the research frame by providing empirical and theoretical evidence that model collapse is not an absolute certainty but is highly dependent on data management strategies. It moves beyond the "doom" narrative by demonstrating that the "accumulate" scenario, which is more aligned with real-world data practices, effectively avoids collapse. Furthermore, it quantifies the impact of data ratios and introduces the concept of different "regimes" (scarce vs. plentiful real data) where synthetic data's utility changes. This offers a nuanced understanding of the technical thresholds and data-mixing ratios that influence model stability, suggesting that strategic data curation and accumulation are key to mitigating collapse, rather than it being an inherent mathematical inevitability. The findings support the idea that model collapse is a "manageable engineering challenge" rather than a fundamental law.
