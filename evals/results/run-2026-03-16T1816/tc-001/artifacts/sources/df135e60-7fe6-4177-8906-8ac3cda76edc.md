# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication:** ICML 2024 Workshop on Foundation Models in the Wild

**Authors:** Matthias Gerstgrasser, Rylan Schaeffer, Apratim Dey, Matthew Hughes, Tomasz Korbak, Rajashree Agrawal, Dhruv Pai, John Hughes, Rafael Rafailov, Henry Sleight, Andrey Gromov, Daniel A. Roberts, Diyi Yang, David Donoho, Sanmi Koyejo

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own synthetic outputs progressively degrade in performance. Previous research often assumed that new synthetic data replaces old real data, leading to collapse. This study, however, explores a more realistic scenario where data accumulates over time. Through empirical experiments with language models, diffusion models, and variational autoencoders, the authors demonstrate that accumulating synthetic data alongside original real data effectively avoids model collapse. This finding is further supported by theoretical analysis using a tractable framework of linear models, which shows that accumulating data leads to a finite upper bound on test error, preventing indefinite degradation.

## Key Claims

*   Training generative models on their own generated outputs can lead to "model collapse," a progressive degradation of performance.
*   Previous studies often assumed data replacement, which exacerbates model collapse.
*   Accumulating synthetic data alongside original real data, rather than replacing it, effectively avoids model collapse.
*   This finding holds across various model types (transformers, diffusion models, VAEs) and data modalities (text, molecules, images).
*   Theoretical analysis of linear models confirms that data accumulation leads to a bounded test error, preventing collapse, whereas data replacement results in linearly increasing test error.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse..."
*   "...we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."
*   In the linear model framework: "if data are replaced across model-fitting iterations, then the test squared error increases linearly with the number of iterations n."
*   In the linear model framework: "if data instead accumulate across model-fitting iterations, then the test squared error is upper bounded by a relatively small constant, meaning model collapse is avoided."

## Contribution to/Challenge to Research Frame

This source directly supports the inferred research frame by providing strong empirical and theoretical evidence that model collapse is **not** an inevitable outcome of recursive training. It challenges the "doom" narrative by identifying a specific mechanism – data accumulation – that mitigates the problem. The paper demonstrates that the "curse of recursion" can be broken by adopting a data management strategy that aligns with real-world data dynamics (accumulation rather than replacement). It offers a technical solution and a more optimistic outlook on the sustainable use of synthetic data in generative model training.
