# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Authors:** Matthias Gerstgrasser, Rylan Schaeffer, Apratim Dey, Rafael Rafailov, Henry Sleight, John Hughes, Tomasz Korbak, Rajashree Agrawal, Dhruv Pai, Andrey Gromov, Daniel A. Roberts, Diyi Yang, David Donoho, Sanmi Koyejo
**Year/Venue:** ICML 2024 Workshop on Foundation Models in the Wild

This paper investigates the phenomenon of model collapse in generative AI, where models trained recursively on their own synthetic outputs degrade in performance and diversity. The authors propose and demonstrate that accumulating synthetic data alongside the original real data effectively avoids model collapse, whereas replacing real data with synthetic data leads to collapse. This finding is supported by empirical evidence across language models, diffusion models, and variational autoencoders, as well as a theoretical analysis using a simplified linear model framework. The core contribution is the demonstration that data accumulation is a robust strategy to mitigate model collapse, challenging the notion that it is an inevitable consequence of recursive training.

## Key Claims

*   Accumulating synthetic data alongside real data prevents model collapse, while replacing real data with synthetic data leads to model collapse.
*   Model collapse is characterized by a worsening of test error over increasing iterations of the model-data loop.
*   A theoretical framework based on linear models provides intuition for why data accumulation avoids model collapse by bounding test error.

## Relevant Evidence & Quotes

*   **Language Models (TinyStories):** "GPT-2 (9M) shows test cross-entropy increasing from 1.82 to 2.39 (+0.58) at iteration 4 when replacing data, but remains at 1.74 (-0.07) when accumulating data."
*   **Diffusion Models (GEOM-Drugs):** Test loss degrades from ~100 to ~300 over 8 iterations when replacing data, but remains relatively constant around ~50 when accumulating data.
*   **Variational Autoencoders (CelebA):** Test loss increases from ~0.020 to ~0.030 over 8 iterations when replacing data, but plateaus around ~0.021 when accumulating data.
*   **Linear Models (Theoretical):** "In a simplified linear regression framework, if data are replaced, the test squared error grows linearly with the number of iterations `n`. If data accumulate, the test squared error is upper bounded by a finite constant, independent of `n`."
*   **Direct Quote:** "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   **Direct Quote:** "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."

## Contribution to Research Frame

This source directly supports the inferred research frame that "Model collapse is a function of statistical approximation errors and variance loss that can be mathematically modeled and empirically delayed through strategic data hygiene and hybrid training regimes."

*   **Supports Mathematical Modeling:** The paper provides both empirical evidence across multiple model types and a theoretical analysis using linear models to explain the phenomenon of model collapse and its mitigation. The theoretical derivation showing a finite upper bound on test error in the accumulation setting aligns with the idea of mathematically modeling and controlling these errors.
*   **Supports Data Hygiene/Hybrid Regimes:** The central finding that *accumulating* (a form of data hygiene/hybrid regime) synthetic data with real data avoids collapse, while *replacing* it leads to collapse, directly validates the importance of how data is managed in recursive training. The distinction between replacement and accumulation highlights a critical "strategic data hygiene" aspect.
*   **Addresses Statistical Approximation Errors:** The paper implicitly addresses statistical approximation errors by showing how they accumulate and lead to divergence in the "replace" scenario, and how this divergence is prevented in the "accumulate" scenario.
*   **Challenges Inevitability:** By demonstrating a robust mitigation strategy (data accumulation), the paper challenges the notion that model collapse is an inevitable "curse of recursion," suggesting it is a manageable engineering challenge under specific conditions.
