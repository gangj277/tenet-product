# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication Info:** Not specified in the provided text.

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own outputs progressively degrade in performance. Previous research often assumed that new synthetic data replaces old real data, leading to collapse. However, this study explores a more realistic scenario where data accumulates over time. The authors empirically demonstrate, across various model types (language models, diffusion models, VAEs) and data modalities (text, molecules, images), that accumulating synthetic data alongside original real data *avoids* model collapse. Theoretically, they extend a linear model framework to prove that accumulating data results in a finite, bounded test error, unlike the linearly increasing error observed when data is replaced. This suggests that data accumulation is a robust strategy against the "curse of recursion."

## Key Claims

*   Training generative models on their own outputs can lead to "model collapse," a progressive degradation of performance.
*   Previous studies often assumed data replacement, which contributes to model collapse.
*   A more realistic scenario involves data accumulation, where new synthetic data is added to existing real data.
*   Empirical experiments across language, molecule, and image generation show that accumulating data avoids model collapse.
*   Theoretical analysis of linear models confirms that data accumulation leads to a bounded test error, preventing collapse.
*   Data accumulation is presented as a robust strategy to mitigate the negative effects of recursive training on generative models.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."
*   Regarding the linear model framework: "if data are replaced across model-fitting iterations, then the test squared error increases linearly with the number of iterations n: $E_{\text{replace}}(w_n) = \frac{\sigma^2 d}{T-d-1} n$".
*   "if data instead accumulate across model-fitting iterations, then the test squared error is upper bounded by a relatively small constant, meaning model collapse is avoided: $E_{\text{accum}}(w_n) < \frac{\sigma^2 d}{T-d-1} \frac{\pi^2}{6}$".
*   "in the data accumulation setting, because iteration $i$ contributes fraction $1/i$ to the training dataset, the additional noise from the $i$th iteration of synthetic data has its effect on the model MSE shrunken proportional to $1/i^2$ (due to squared error). The summability of $1/i^2$ prevents the test error from growing indefinitely."
