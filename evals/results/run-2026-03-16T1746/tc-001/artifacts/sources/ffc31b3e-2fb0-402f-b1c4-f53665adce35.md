# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication:** ICML 2024 Workshop on Foundation Models in the Wild

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own outputs progressively degrade in performance. Previous work largely assumed that new synthetic data replaces old data, leading to collapse. However, this study explores a more realistic scenario where data accumulates over time. The authors empirically demonstrate across various generative models (language models, diffusion models, VAEs) and datasets that accumulating synthetic data alongside original real data *avoids* model collapse. Theoretically, they extend a linear model framework to prove that accumulating data results in a finite upper bound on test error, independent of the number of iterations, thus preventing collapse. The findings suggest that data accumulation is a robust strategy against model collapse.

## Key Claims

*   Training generative models on their own generated outputs can lead to performance degradation, a phenomenon termed "model collapse."
*   Previous studies often assumed data replacement, where new synthetic data replaces old data, which tends to cause model collapse.
*   This paper empirically and theoretically shows that *accumulating* synthetic data alongside original real data avoids model collapse.
*   This finding holds across different model architectures (transformers, diffusion models, VAEs), data types (text, molecules, images), and hyperparameters.
*   A theoretical analysis using a linear model framework proves that accumulating data leads to a finite, bounded test error, irrespective of the number of training iterations, thus preventing unbounded degradation.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "We obtain similar results for deep generative models on other types of real data: diffusion models for molecule conformation generation and variational autoencoders for image generation."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."
*   Figure 2, 3, and 4 show that cross-entropy validation loss increases when repeatedly replacing data but not when accumulating data for language modeling, molecular conformation generation, and image generation, respectively.
*   Figure 18 demonstrates that in a linear regression setting, test error grows linearly with iterations when data is replaced, but has a finite upper bound when data accumulates.

## Contribution to Research Frame

This source directly addresses the core question of the research frame: "Is model collapse an inevitable consequence of recursive synthetic training, or can it be managed?" The paper provides strong empirical and theoretical evidence *against* the inevitability of model collapse, specifically under the condition of data accumulation.

It challenges the implicit assumption in some prior work that model collapse is a fundamental limitation. Instead, it posits that the *method* of data handling (replacement vs. accumulation) is a critical factor. The findings suggest that the "curse of recursion" can be broken by adopting a data accumulation strategy, which aligns with the research frame's interest in identifying "technical boundary conditions" and "manageable engineering challenges." The paper's theoretical proof offers a mathematical basis for why accumulation prevents collapse, contributing to understanding the "specific mathematical mechanisms" and offering a potential "mitigation strategy." This work directly supports the hypothesis that strategic data integration can maintain model stability.
