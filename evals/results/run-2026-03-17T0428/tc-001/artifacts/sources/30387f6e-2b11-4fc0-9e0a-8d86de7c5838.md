# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication Info:** Not specified in the provided text.

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own outputs progressively degrade in performance. Previous research often assumed that new synthetic data replaces old real data, leading to collapse. However, this study explores a more realistic scenario where data accumulates over time. Through experiments with language models, diffusion models for molecules, and variational autoencoders for images, the authors demonstrate that accumulating synthetic data alongside original real data *avoids* model collapse. Theoretically, they extend a linear model framework to prove that accumulating data results in a finite upper bound on test error, thus preventing collapse, unlike the unbounded error seen when data is replaced.

## Key Claims

*   Training generative models on their own outputs can lead to "model collapse," a progressive degradation of performance.
*   Previous studies often assumed data replacement, which contributes to model collapse.
*   A more realistic scenario involves data accumulation, where new synthetic data is added to existing real data.
*   Empirically, accumulating synthetic data with real data avoids model collapse across various model types (transformers, diffusion models, VAEs) and data modalities (text, molecules, images).
*   Theoretically, in a linear model framework, accumulating data leads to a finite upper bound on test error, preventing collapse, whereas data replacement results in unbounded error.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."
*   In the linear model section: "if data are replaced across model-fitting iterations, then the test squared error increases linearly with the number of iterations n... if data instead accumulate across model-fitting iterations, then the test squared error is upper bounded by a relatively small constant, meaning model collapse is avoided."

## Contribution to/Challenge of Research Frame

This source directly addresses the core question of whether model collapse is inevitable. It challenges the implicit assumption in some prior work that recursive training *necessarily* leads to collapse by introducing and validating the "data accumulation" scenario.

*   **Contribution:** It provides strong empirical and theoretical evidence that model collapse is *not* inevitable under a specific, more realistic data handling regime (accumulation). It offers a clear mitigation strategy: mixing new synthetic data with existing real data.
*   **Challenge:** It challenges the "curse of recursion" narrative by showing that the problem might be more about *how* data is managed (replacement vs. accumulation) rather than an inherent mathematical certainty of performance degradation in all recursive training loops. It suggests that the "tipping point" might be avoided by maintaining a diverse and growing dataset.
