# An Upper Bound on the Loss from Approximate Optimal-Value Functions

**Publication:** Technical Note

## Summary

This technical note derives an upper bound on the performance loss incurred when using an approximate optimal-value function with a greedy policy in Markov Decision Processes (MDPs). The authors present a slightly tighter bound than previously established and extend this analysis to Q-learning. The core finding is that small errors in approximating the optimal value function do not necessarily lead to catastrophic performance degradation, providing theoretical justification for the use of function approximation in reinforcement learning. The note also discusses the implications of approximate payoffs and the practical challenges of computing these bounds.

## Key Claims

*   Small errors in approximating an optimal value function (V) with a greedy policy (πV) result in a bounded performance loss.
*   The derived upper bound on this loss is $\frac{2\gamma\varepsilon}{1 - \gamma}$, where $\gamma$ is the discount factor and $\varepsilon$ is the maximum approximation error in the value function.
*   This bound is tighter than previous results by a factor of $\gamma$.
*   The analysis can be extended to cases with approximate payoffs, where the loss bound increases.
*   For Q-learning, a similar bound of $\frac{2\varepsilon}{1 - \gamma}$ is established, where $\varepsilon$ is the approximation error in the Q-function.
*   The practical computation of these bounds is challenging as it requires knowledge of the optimal value function, but an alternative computable bound can be derived using the maximum difference between successive value function approximations.
*   The theoretical guarantees hold under specific conditions: infinite-horizon discounted tasks, good value function approximations, greedy policy selection, and discount factors not too close to 1.

## Relevant Evidence/Quotes

*   "This note presents a result from Bertsekas (1987) that guarantees small errors in approximating a task's optimal value function will not lead to catastrophic performance degradation when actions are selected by a greedy policy."
*   "We derive a slightly tighter upper bound on performance loss than Bertsekas (1987) and extend it to Q-learning (Watkins, 1989)."
*   **THEOREM:** "If V is a function such that for all x ∈ X, |V*(x) - V(x)| ≤ ε, and πV is a greedy policy for V, then for all x: $L_V(x) \leq \frac{2\gamma\varepsilon}{1 - \gamma}$"
*   **COROLLARY 2:** "If |Q*(x, a) - Q(x, a)| ≤ ε, for all x ∈ X and a ∈ A(x), then for all x ∈ X: $L_Q(x) \leq \frac{2\varepsilon}{1 - \gamma}$"
*   "The theorem and its corollaries guarantee that for infinite-horizon discounted payoffs, DP-based learning approaches will not perform far from optimally if (a) good approximations to optimal value functions are achieved, (b) a corresponding greedy policy is followed, and (c) the discount factor, γ, is not too close to 1.0."

## Contribution to Research Frame

This source directly supports the research frame by providing a theoretical foundation for understanding performance degradation due to approximation errors in value functions. The core theorem establishes that such errors do not necessarily lead to "catastrophic" failure, aligning with the frame's assumption that model collapse is conditional rather than inevitable.

Specifically:

*   **Mathematical Drivers of Error:** The paper mathematically defines and bounds the "loss" ($L_V(x)$) resulting from approximation errors ($\epsilon$). This directly addresses the subquestion about the specific mathematical drivers of "approximation error" versus "functional error." The bound $\frac{2\gamma\varepsilon}{1 - \gamma}$ quantifies how approximation error translates to performance loss.
*   **Conditional Failure:** The existence of a bounded loss, rather than unbounded degradation, supports the idea that collapse is conditional. The bound is dependent on $\epsilon$ (the approximation quality) and $\gamma$ (the discount factor), suggesting that controlling these factors can prevent severe performance drops.
*   **Mitigation Strategies (Implicit):** While not proposing specific mitigation techniques, the work implies that improving the approximation quality ($\epsilon$) or adjusting the discount factor ($\gamma$) are theoretical avenues to mitigate loss. This aligns with the frame's interest in evaluating mitigation strategies.
*   **Distributional Shift (Implicit):** The concept of an approximate value function and the resulting policy implicitly relates to distributional shift. If the approximate value function deviates significantly from the true optimal one, the resulting policy will lead the agent to different states or take different actions, thus shifting the data distribution experienced by the learning algorithm. The bounds suggest this shift is manageable if $\epsilon$ is small.

The source challenges the frame by highlighting that "small losses" in discounted payoffs might not align with intuitive notions of success, particularly in tasks where specific outcomes (like reaching a goal) are critical and might be poorly represented by small value differences. This suggests that the *nature* of the approximation error and the task formulation are crucial, not just the magnitude of $\epsilon$.
