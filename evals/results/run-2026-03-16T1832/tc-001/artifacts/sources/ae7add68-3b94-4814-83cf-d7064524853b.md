# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication:** ICML 2024 Workshop on Foundation Models in the Wild

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own outputs progressively degrade in performance. Previous research often assumed that new synthetic data replaces old data, leading to collapse. However, this study explores a more realistic scenario where data accumulates over time. Through experiments with language models, diffusion models for molecule generation, and VAEs for image generation, the authors demonstrate that accumulating synthetic data alongside real data effectively avoids model collapse. This finding is supported by a theoretical framework using linear models, which shows that accumulating data leads to a bounded test error, unlike the unbounded error seen when data is replaced.

## Key Claims

*   Training generative models on their own outputs can lead to "model collapse," characterized by progressively degrading performance.
*   Previous studies often assumed data replacement, which exacerbates model collapse.
*   Accumulating synthetic data alongside original real data effectively avoids model collapse across various model types (language, diffusion, VAEs) and data modalities (text, molecules, images).
*   A theoretical framework with linear models confirms that data accumulation results in a bounded test error, preventing indefinite degradation.
*   The "curse of recursion" may be mitigated by accumulating synthetic data rather than solely relying on it to replace real data.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "We find that replacing data yields model collapse for all models and all datasets, whereas accumulating data avoids model collapse."
*   In the context of linear models:
    *   Data Replacement: $E_{test}^{Replace}(\\hat{w}_n) = \\frac{\\sigma^2 d}{T-d-1} \\times n$ (test error increases linearly with iterations $n$)
    *   Data Accumulation: $E_{test}^{Accum}(\\hat{w}_n) \\leq \\frac{\\sigma^2 d}{T-d-1} \\times \\frac{\\pi^2}{6}$ (test error has a finite upper bound)

## Contribution to Research Frame

This source directly addresses the core concern of the research frame regarding the inevitability of model collapse due to recursive training on synthetic data. It challenges the notion that collapse is an inherent mathematical certainty by providing strong empirical and theoretical evidence that **data accumulation is a viable strategy to avoid collapse**.

*   **Challenges "inevitable" aspect:** The paper's central finding is that model collapse is *not* inevitable if data accumulates. This directly counters the idea that degradation is a guaranteed outcome.
*   **Focus on "manageable engineering challenge":** By demonstrating that a specific data management strategy (accumulation) prevents collapse, it reframes the problem as an engineering challenge rather than a fundamental limitation.
*   **Interplay of synthetic data density and preservation:** The research implicitly supports the idea that the *way* synthetic data is integrated (accumulated vs. replaced) is critical. Accumulation allows for a continuous presence of original data, potentially preserving "tail" distributions better than replacement.
*   **Efficacy of mitigation strategies:** Data accumulation is presented as a key mitigation strategy, empirically validated across multiple domains.

This source provides crucial empirical and theoretical backing for the hypothesis that model collapse can be avoided, shifting the focus from the inevitability of collapse to the effectiveness of data management strategies.
