# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication:** ICML 2024 Workshop on Foundation Models in the Wild

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own synthetic outputs progressively degrade in performance. Previous studies often assumed that new synthetic data replaces old real data, leading to collapse. However, this research explores a more realistic scenario where data accumulates over time. The authors empirically demonstrate across various generative models (language, diffusion, VAEs) and theoretically, using a linear model framework, that accumulating synthetic data alongside original real data *avoids* model collapse. This is because the accumulating data provides a finite upper bound on test error, preventing indefinite degradation.

## Key Claims

*   Training generative models on their own outputs can lead to "model collapse," a progressive degradation of performance.
*   Previous studies often assumed data replacement, which exacerbates model collapse.
*   Accumulating synthetic data with original real data, rather than replacing it, effectively avoids model collapse.
*   This finding holds true across different model architectures (transformers, diffusion models, VAEs) and data modalities (text, molecules, images).
*   A theoretical framework using linear models confirms that data accumulation leads to a finite upper bound on test error, preventing collapse, whereas data replacement results in linearly increasing test error.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   "Our work provides consistent empirical and theoretical evidence that data accumulation avoids model collapse."

## Contribution to Research Frame

This source directly addresses and refines the inferred research frame by providing strong evidence that model collapse is *not* an inherent law of recursive training. Instead, it demonstrates that the *method* of data handling (accumulation vs. replacement) is a critical factor. The paper's findings support the idea that strategic integration of real-world data (through accumulation) can mitigate the risks of synthetic data loops, aligning with the frame's emphasis on data hygiene and the ratio of real-to-synthetic data. The theoretical analysis also contributes to understanding the mathematical drivers of this phenomenon, specifically how accumulation leads to bounded error.
