# Is Model Collapse Inevitable? Breaking the Curse of Recursion by Accumulating Real and Synthetic Data

**Publication:** ICML 2024 Workshop on Foundation Models in the Wild
**Authors:** Matthias Gerstgrasser, Rylan Schaeffer, Apratim Dey, Rafael Rafailov, Henry Sleight, John Hughes, Tomasz Korbak, Rajashree Agrawal, Dhruv Pai, Andrey Gromov, Daniel A. Roberts, Diyi Yang, David Donoho, Sanmi Koyejo

## Summary

This paper investigates the phenomenon of "model collapse," where generative models trained on their own outputs progressively degrade in performance. Previous research often assumed that new synthetic data replaces old data, leading to collapse. However, this work explores a more realistic scenario where data accumulates over time. The authors empirically demonstrate, across various model types (transformers for language, diffusion models for molecules, VAEs for images) and theoretically using a linear model framework, that accumulating synthetic data alongside original real data *avoids* model collapse. This is because the test error in the accumulating data scenario has a finite upper bound, unlike the linearly increasing error seen when data is replaced. The findings suggest that data accumulation is a robust strategy against model collapse.

## Key Claims

*   Training generative models on their own outputs can lead to a phenomenon called "model collapse," where performance degrades over iterations.
*   Previous studies often assumed data replacement, which tends to cause model collapse.
*   A more realistic scenario where data accumulates (new synthetic data is added to existing real data) avoids model collapse.
*   This finding holds true empirically for transformers, diffusion models, and VAEs across different data modalities.
*   Theoretically, in a linear model framework, accumulating data leads to a finite upper bound on test error, preventing collapse, whereas data replacement leads to unbounded error.

## Relevant Evidence/Quotes

*   "We confirm that replacing the original real data by each generation's synthetic data does indeed tend towards model collapse, then demonstrate that accumulating the successive generations of synthetic data alongside the original real data avoids model collapse."
*   "Previous work used this framework to show that if data are replaced, the test error increases with the number of model-fitting iterations; we extend this argument to prove that if data instead accumulate, the test error has a finite upper bound independent of the number of iterations, meaning model collapse no longer occurs."
*   In the data accumulation setting, "the additional noise from the ith iteration of synthetic data has its effect on the model MSE shrunken proportional to 1/i² (due to squared error). The summability of 1/i² prevents the test error from growing indefinitely."

## Contribution to Research Frame

This source directly supports the inferred research frame that "Model collapse is a manageable risk rather than a fundamental law of generative AI, contingent upon the ratio of 'fresh' real-world data to synthetic data and the specific sampling methods used."

Specifically, it provides strong empirical and theoretical evidence *against* the inevitability of model collapse. The key differentiator identified is the data handling strategy: **accumulation** (mixing real and synthetic) versus **replacement** (synthetic only). This aligns with the frame's emphasis on data composition strategies. The paper's findings that accumulating data avoids collapse, and that the test error is bounded, directly counters the idea that recursive training is inherently doomed. It suggests that the "curse of recursion" can be broken by adopting specific data management practices, reinforcing the idea that collapse is a *contingent* risk, not an absolute one.
